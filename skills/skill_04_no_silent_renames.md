# Skill 04 — No Silent Renames

## Rule
Renaming is a breaking change.

## Requirements
A rename requires:
- Explicit user approval
- Entry in `/docs/name_change_log.md`
- Repo-wide consistency

## Prohibited
- Silent renames
- Transitional aliases
- “Just updating spelling”

## Rationale
Silent renames cause cascading failures and invalidate frozen modules.
