# Skill 03 â€” No New Names Without Permission

## Rule
The coder MUST NOT introduce any new public or private names without approval.

## Includes
- Files
- Classes
- Methods
- Fields
- Providers
- Database schema
- Events
- Enums

## Required Action
If a new name is required, the coder must STOP and request permission.

## Required Request Format
- Need
- Proposed name(s)
- Scope
- Alternatives considered

## Enforcement
Any unapproved new name invalidates the change.
