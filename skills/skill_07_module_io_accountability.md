## Rule
Every module must explicitly document its inputs and outputs.

## Requirements
- `/docs/module_io_registry.md` entry required
- Inputs must be typed
- Outputs must be typed
- Changes must update the registry

## Rationale
If data flow is undocumented, it is untrusted.

A module cannot be frozen without IO documentation.
